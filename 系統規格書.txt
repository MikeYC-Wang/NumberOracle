# 系統規格書 (PRD)：NumberOracle 綜合分析與預測系統

## 1. 專案概述 
本專案為一個基於「純歷史數據統計與機率運算」的彩券分析平台。系統需自動化抓取歷史開獎數據，透過關聯式資料庫進行深度聚合運算，並以高互動性的現代化網頁介面提供使用者冷熱門號碼、遺漏值分析及數據導向的選號建議。

首波實作目標彩種：
1. 今彩539 (Daily Cash) - 5個號碼
2. 大樂透 (Lotto 6/49) - 6個號碼 + 1個特別號
3. 威力彩 (SuperLotto 6/38) - 第一區6個號碼 + 第二區1個特別號

## 2. 技術架構 
* 前端: Vue.js 3 (Composition API) + Vite + TypeScript
* 後端: Python 3 + Django + Django REST Framework (DRF)
* 資料庫: PostgreSQL (資料庫名稱已建立為 NumberOracle)
* 視覺化特效: Apache ECharts、Spline 3D
* UI/UX 規範: 全面採用 RWD 響應式設計，圖示統一使用 FontAwesome。全站嚴禁使用任何 Emoji。

## 3. 視覺與 UI 設計規範
* 整體風格：方案 B「清爽便當風」，呈現輕鬆日常、文青感。台式微復古 + 現代扁平化 (Modern Flat)，結合台灣小吃插畫風格圖片與手繪插畫感 Logo。
* 色彩計畫：
    * 背景色：#F4F1DE (暖白/米黃) - 提供乾淨、低對比閱讀環境。
    * 主色調：#E9C46A (炸排骨黃) - 用於 Navbar、主要按鈕與 3D 互動元素。
    * 輔助色：#2A9D8F (燙青菜綠) - 用於標示冷門號碼回補、走勢上升等正向數據標籤。
    * 強調色：#E76F51 (辣椒紅) - 用於標示熱門連莊號、警示訊息或重要中獎通知。

## 4. 資料庫架構設計準則 (Backend)
請依據上述三種目標彩券的特性，設計出高擴充性、高查詢效能的 PostgreSQL 資料庫關聯結構。
1. 需包含遊戲種類主表與開獎紀錄明細表。
2. 必須妥善處理「依落球順序」與「依大小順序」兩種號碼陣列。
3. 強烈建議使用 PostgreSQL 的 INTEGER[] 陣列儲存格式，並配合 GIN 索引或 Window Functions 來優化統計查詢。
4. 提出 Schema 後，需簡述如何透過該結構高效查詢「冷熱門號碼」與「遺漏值」。

## 5. 核心功能模組
* 自動化爬蟲與資料流 (Backend): 撰寫腳本解析台彩內部 API 獲取歷史與最新資料，需實作防呆與限流機制 (避免 IP 封鎖)，並封裝為 Django Management Command 以利排程。
* 統計分析 RESTful API (DRF): 提供歷史清單、冷熱門統計、遺漏值統計 API。
* 前端介面模組 (Vue 3): 實作總覽儀表板、彩種專屬分析頁 (ECharts 圖表)，以及結合 Spline 3D 物理慣性特效的預測搖獎區。
